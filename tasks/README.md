# Tasks - DevinciWatch Scrum Board

Ce dossier contient l'organisation opérationnelle de l'équipe DevinciWatch, basée sur le [Gestion de projet (07)](../documents/07_gestion_de_projet/rendu_principal.md).

## Structure

- `README.md` - Ce fichier (organisation Scrum)
- `EPIC-XX-nom.md` - Détail de chaque Epic avec User Stories et Tasks
- `team-assignments.md` - Répartition des responsabilités par membre
- `sprint-backlog.md` - Suivi du sprint en cours

## Utilisation

1. **Chaque Epic** a son fichier détaillé avec :
   - Lien vers §8 du Gestion de projet
   - Liste des User Stories (ID, Story, SP, Sprint)
   - Checklist des Tasks techniques (liées aux issues GitHub)
   - Définition of Ready et Done spécifique

2. **Team Assignments** précise qui travaille sur quoi selon le Gestion de projet §4

3. **Sprint Backlog** est mis à jour à chaque Sprint Planning

## Workflow

1. Créer une Issue GitHub avec le template `user-story.md`
2. Lier l'issue au Epic correspondant (milestone ou label)
3. Découper en Tasks techniques (issues `task.md`)
4. Assigner selon `team-assignments.md`
5. Déplacer dans le Scrum board au Sprint Planning
6. Mettre à jour le fichier Epic correspondant
